# MindDump — спецификация

Цифровая реализация метода **Bullet Journal** (Ryder Carroll). Основной принцип — **осознанность вместо автоматизации**: приложение помогает фиксировать и не теряет данные, но не двигает их за пользователя. Все важные переходы — ручные, все автоматические — только по датам.

---

## 1. Экраны

Навигация (порядок): **Сегодня → Неделя → Месяц → Полгода → Коллекции → Настройки**

| Экран | Роль | BuJo-аналог |
|---|---|---|
| **Сегодня** | Daily Log — текущий день | Daily Log |
| **Неделя** | 7 дней Пн–Вс, опциональный, включается в Настройках | — (расширение) |
| **Месяц** | Календарь месяца с бейджами, блок «Заметки месяца», блок «Задачи без даты», ритуал migration | Monthly Log |
| **Полгода** | 6 месяцев вперёд, по-месячные блоки | Future Log |
| **Коллекции** | Тематические списки заметок | Collections |
| **Настройки** | Данные, тумблеры, пороги | — |

### Настройки (V1)
- Тумблер: **Показывать экран «Неделя»** (по умолчанию выкл)
- Первый день недели: **Пн** / Вс (по умолчанию Пн)
- Формат времени: **24ч** / 12ч (по умолчанию 24ч)
- Порог «Пора разобрать заметки»: **50** (целое число ≥ 10)
- Экспорт / импорт / очистка данных

## 2. Типы записей

Три взаимопревращаемых типа:

| Тип | Символ | Смысл |
|---|---|---|
| **Задача** | `·` (точка) | Что-то, что нужно сделать. Имеет состояние. |
| **Событие** | `○` | Факт, привязанный к конкретному моменту времени (произойдёт или произошёл). |
| **Заметка** | `—` | Мысль, наблюдение, идея. Не требует действия. |

### Общие поля
```
Entry {
  id                     uuid
  type                   "task" | "event" | "note"
  text                   string
  month                  YYYY-MM        // всегда есть
  date                   YYYY-MM-DD | null
  created_at             timestamp
  updated_at             timestamp      // обновляется при любом изменении
  addenda                Addendum[]     // цепочка дополнений
}

Addendum {
  id                     uuid
  text                   string (≤ 280 символов)
  added_at               timestamp
}
```

### Особые поля по типам

**Task**
```
status                   "open" | "done" | "forgotten"
```

**Event**
```
time                     HH:MM | null   // 24ч, опционально
status                   "upcoming" | "past" | "forgotten"
// past считается автоматически по date vs now; не хранится в БД
```
- `date` может быть `null` **только пока `month ∈ Полгода`** (планы без точного дня).
- Когда наступает соответствующий месяц, событие переходит в блок «Без даты» на экране Месяц, пока пользователь не укажет день.

**Note**
```
collection_id            uuid | null    // null = в «Заметках месяца»
status                   "active" | "forgotten"
```

## 3. Правила видимости (все автоматические — только по дате)

Нет «перетекания» записи между контейнерами. Запись **живёт в одной координате во времени** (`month` + опц. `date`), экраны лишь **показывают срез** этой базы по своим фильтрам.

### Сегодня показывает
- `type ∈ {task, event}`: `date = сегодня` и `status = open | upcoming`
- `type = note`: `date = сегодня` и `collection_id = null`
- События сортируются по `time` (раньше сверху, без времени — в конце блока событий)

### Неделя (если включена)
- Всё из текущей недели (Пн–Вс): записи группируются по дням
- Правила фильтрации те же, что у Сегодня, но для 7 дней

### Месяц показывает
- **Календарная сетка** с бейджами у каждого числа: `N• M○ K—` (счётчик задач / событий / заметок этого дня, включая заметки с коллекцией — они считаются для правды дня)
- **Клик по числу** — раскрывает список дня **на месте**, без перехода в Сегодня
- **Блок «Заметки месяца»** — плоский список заметок этого месяца с `collection_id = null`, без группировки по дням
- **Блок «Без даты»** — задачи и события этого месяца с `date = null`
- **Блок «Итоги месяца»** — появляется за 2 дня до конца месяца (см. раздел 5)

### Полгода показывает
- 6 месяцев от текущего вперёд
- Внутри каждого месяца — задачи и события этого `month` (включая с `date = null`)
- **Заметки не показываются никогда** (заметки — про прошлое/настоящее, не про будущее)

### Коллекции показывает
- Список всех коллекций + счётчик заметок в каждой
- При открытии коллекции — заметки этой коллекции, новые сверху

### Сегодня = пустой день?
Показываем подсказку «Запишите что-нибудь» + кнопка «+».

## 4. Создание записи

Кнопка «+» доступна на всех журнальных экранах. По умолчанию `date`/`month` подставляются из контекста:

| Экран создания | Значение `date` и `month` по умолчанию |
|---|---|
| Сегодня | `date = сегодня` |
| Неделя, клик по дню | `date = этот день` |
| Месяц, клик по дню | `date = этот день` |
| Месяц, блок «Без даты» | `date = null`, `month = текущий месяц` |
| Месяц, блок «Заметки месяца» | `date = сегодня` (заметка всегда к дню), `collection_id = null` |
| Полгода, клик по месяцу | `month = этот`, `date = null` (пользователь может уточнить день) |
| Коллекции, внутри коллекции | `date = сегодня`, `collection_id = эта коллекция` |

Пользователь может изменить любое поле в диалоге создания.

### Заметка: выбор коллекции при создании
В диалоге создания заметки — селектор:
- **Без коллекции** (по умолчанию)
- Любая существующая коллекция
- **Создать новую…** — ввод имени с проверкой дубликата (case-insensitive, trim). Если имя занято — «Такая коллекция уже есть, добавить туда?»

## 5. Migration — ритуал «Пора подвести итоги»

### Триггер
За **2 дня до конца месяца** на экране Месяц появляется баннер «📋 Пора подвести итоги». Баннер не пропадает сам — висит, пока в блоке «Итоги месяца» есть неразобранные записи, **даже если уже 2-е или 10-е число нового месяца**. 1-е число — моральный крайний срок, дальше баннер становится напоминанием о долге.

### Блок «Итоги месяца»
Два подраздела, плоскими списками (без группировки по дням):

**Незакрытые задачи месяца** (`type = task`, `status = open`, `month = прошедший`)

Действия на карточке задачи:
- `›` **В следующий месяц** → выбор: конкретный день / «Без даты»
- `‹` **В Полгода** → выбор месяца (в пределах 6 мес.), `date = null`
- `⇄` **В Событие** → обязательный выбор даты + опц. времени
- `⇄` **В Заметку** → выбор: оставить в блоке месяца / в коллекцию
- `~~` **Забыть** → `status = forgotten`
- `✓` **Выполнено** (если вдруг вспомнил, что сделал) → `status = done`

**Заметки без коллекции** (`type = note`, `collection_id = null`, `month = прошедший`)

Действия на карточке заметки:
- `⇄` **В задачу** → выбор: конкретный день / «Без даты»
- `⇄` **В Событие** → дата + опц. время
- `→` **В коллекцию** → выбор/создание коллекции
- `~~` **Забыть**
- (не трогать) — заметка **автоматически переедет** в «Заметки месяца» следующего месяца (см. ниже)

### Автомиграция неразобранных заметок
В отличие от задач, неразобранные заметки **не висят бесконечно**:
- В момент создания следующего месяца заметки прошлого с `collection_id = null` меняют `month` на новый.
- Это значит, что на экране Месяц они начинают появляться в блоке «Заметки месяца» уже нового месяца.
- Дата создания (`created_at`) и `date` **не меняются** — для истории.

### Порог переполнения заметок
Независимо от migration:
- Счётчик = все заметки в базе с `collection_id = null` и `status = active`
- Если `count ≥ threshold` (по умолчанию 50) — отдельный баннер **«Пора разобрать заметки»** на экране Месяц
- Висит до опускания ниже порога или до нажатия «Отложить» (скрывает на 7 дней)

### События в migration
Не участвуют. Прошедшие события остаются на своих днях в истории Месяца/Недели. Хочешь «передвинуть» — либо создай новое событие, либо конвертируй в задачу.

## 6. Действия над записями (обычный режим, не migration)

### Задача в Сегодня / Неделе
- `✓` **Выполнено** → `status = done` (зачёркивание на карточке)
- `›` **Перенести** → выбор дня **до конца текущего месяца** (завтра / конкретный день). Дальше текущего месяца не даёт — это сознательное ограничение (ушедшие в следующий месяц задачи должны пройти через ритуал).
- `⇄` **Изменить тип** → Событие (дата + опц. время) / Заметка
- `+ Дополнить` → см. раздел 7
- `✎ Редактировать` → меняет `text` (с подтверждением, если запись уже создана и имеет `updated_at > created_at`)
- `~~ Забыть` / `🗑 Удалить`

### Задача в Месяце (клик по карточке в раскрытом дне)
Те же действия. `Перенести` доступен только внутри текущего месяца. Для переноса в другой месяц пользователь ждёт migration.

### Событие
- `⇄` Изменить тип → Задача / Заметка
- `+ Дополнить`
- `✎ Редактировать`
- `~~ Забыть` / `🗑 Удалить`
- Прошедшее событие — те же действия + осталось на своём дне как история

### Заметка в блоке «Заметки месяца» / Сегодня / Неделе
- `⇄` В задачу → выбор дня / «Без даты»
- `⇄` В событие → дата + опц. время
- `→` В коллекцию → выбор/создание
- `+ Дополнить`
- `✎ Редактировать`
- `~~ Забыть` / `🗑 Удалить`

### Заметка внутри коллекции
- `↩ Убрать из коллекции` → `collection_id = null`, вернётся в «Заметки месяца» своего `month`
- `→ В другую коллекцию`
- Остальные действия те же

### Действия над коллекцией (экран Коллекции)
- `✎` Переименовать (с проверкой дубликатов)
- `🎨` Сменить цвет (если в V1 решим делать; иначе пропускаем)
- `🗑` Удалить → диалог: **«Удалить вместе с N заметками»** / **«Оставить заметки без коллекции (переедут в «Заметки месяца»)»** / Отмена

## 7. Дополнения (addenda / thread)

У каждой записи есть кнопка **`+ Дополнить`** — добавляет новый блок текста с временной меткой под оригиналом. Это не редактирование, а приписка.

### Правила
- Доступно на **любой** карточке везде, где она видна (Сегодня / Неделя / Месяц / Коллекция / migration-экран).
- **Лимит на дополнение: 280 символов** (жёсткий, с счётчиком).
- Оригинал можно **редактировать** — но если запись уже была сохранена, показывается подтверждение («Изменить существующий текст?»).
- Дополнения можно **редактировать и удалять**.
- Дополнения **не меняют видимость** записи — она остаётся в своих координатах во времени. Хочешь «поднять» старую запись — создай новую и сошлись на старую.
- В migration и во всех списках цепочка дополнений **видна** (может быть свёрнута «+3 дополнения» при длине > 1).

### Визуальный пример
```
— Идея для подкаста про BuJo          01.11 14:32
  + нашёл два примера, см. ссылки      02.11 09:15
  + объединить с идеей про планирование 15.11 22:00
```

## 8. Коллекции (V1)

- Простой **плоский список заметок** (только заметки; фото/стикеры/вложения — вне V1).
- Поля: `id`, `name` (уникальное), `created_at`, опц. `color`.
- Создание: на экране Коллекции или налету при создании/преобразовании заметки.
- Удаление: см. раздел 6.
- Заметка может быть **только в одной коллекции** одновременно. Переместить — `→ В другую коллекцию` (атомарно).

---

## 9. Состояние «Забыто»

Запись со `status = forgotten` (для задач и заметок) или «удалена» (hard delete):
- **Забыть (soft)** — остаётся в БД с визуальным зачёркиванием/серостью, не показывается в активных списках, но видна при перелистывании истории (Месяц прошлого).
- **Удалить (hard)** — полностью удаляется из БД. Это действие в «Опасной зоне» меню, с подтверждением.

По умолчанию в UI предлагается «Забыть» — как в бумажном BuJo (зачёркнутое всё равно остаётся видимым). «Удалить» — только для явных ошибок/тестовых записей.

## 11. Что НЕ делаем в V1

Чтобы сохранить суть метода и не превратить приложение в ещё один Todoist:

- ❌ **Повторяющиеся задачи** — каждая задача создаётся осознанно. Повторы — ручное перенесение через migration.
- ❌ **Push-уведомления и напоминания** — журнал, а не ассистент. Пользователь сам открывает приложение.
- ❌ **Длительность событий** — событие это точка во времени.
- ❌ **Авто-перемещение задач между днями/месяцами** — только ручное, кроме наступления даты.
- ❌ **Один-к-многим коллекции** — заметка в одной коллекции одновременно.
- ❌ **Фото, файлы, стикеры в коллекциях**.
- ❌ **«Поднять задачу в месяц» из Сегодня** — между горизонтами только через migration или создание с другой датой.
- ❌ **Синхронизация, облако, мульти-устройство** — локально (можно экспорт/импорт JSON).

## 12. Сигнификаторы (?) — обсудить отдельно

В текущем `index.html` есть поля «Приоритет» (`*`) и «Озарение» (`!`) — это классические BuJo signifiers. В этой спецификации они **пока не описаны**, потому что не обсуждались. Возможные варианты:
- Убрать из UI полностью (меньше концепций)
- Оставить как визуальные метки (звёздочка/восклицание на карточке) без отдельной логики
- Расширить до фильтра «только приоритеты» в Сегодня
